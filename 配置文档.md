# 在新电脑从零跑起 IDA Pro MCP 的完整流程（Windows 优先）

本文档面向第一次在新电脑上搭建本仓库的人，目标是从 0 到可用：
- 能启动 IDA Pro MCP 服务
- 能被 MCP 客户端（如 VS Code）连接
- 能执行基本测试与排障

## 1. 这套工程是什么

本仓库是 IDA Pro MCP Server，作用是把 IDA Pro / idalib 能力暴露为 MCP 工具，供外部客户端（编辑器、Agent、脚本）调用。

核心运行方式有两种：
- IDA 插件模式（推荐）：在 IDA GUI 中启动 MCP 服务
- idalib 无头模式：不打开 IDA GUI，直接对二进制进行服务化分析

## 2. 先决条件

请先准备：
- Windows 10/11（Linux/macOS 也可，命令略有差异）
- Git
- Python 3.11+
- uv（包管理与运行工具）
- IDA Pro 8.3+（推荐 9.0），且有可用授权

注意：IDA Free 不支持此项目。

## 3. 安装基础工具

### 3.1 安装 Git

从官方安装后，确认：

```powershell
git --version
```

### 3.2 安装 Python 3.11+

安装后确认：

```powershell
python --version
```

### 3.3 安装 uv

PowerShell 可用：

```powershell
irm https://astral.sh/uv/install.ps1 | iex
```

确认：

```powershell
uv --version
```

## 4. 拉取仓库并初始化环境

```powershell
git clone <你的仓库地址> ida-pro-mcp
cd ida-pro-mcp
uv sync
```

说明：
- `uv sync` 会按 `pyproject.toml` 与 `uv.lock` 安装依赖
- 如果你不想依赖全局 Python，可直接使用 `uv` 管理环境，不必手动建 venv

可选验证：

```powershell
uv run python --version
```

## 5. 运行方式 A：IDA 插件模式（推荐）

这是最常见方式，适合交互式逆向分析。

### 5.1 安装 IDA 插件

在仓库根目录执行：

```powershell
uv run ida-pro-mcp --install
```

如需卸载：

```powershell
uv run ida-pro-mcp --uninstall
```

### 5.2 启动 IDA 并启动 MCP 服务

1. 打开 IDA，加载目标二进制
2. 在插件菜单中启动 MCP 服务（通常监听 `127.0.0.1:13337`）

默认 MCP URL 常见为：

```text
http://127.0.0.1:13337/mcp
```

### 5.3 端口连通性检查

```powershell
Test-NetConnection 127.0.0.1 -Port 13337
```

若 `TcpTestSucceeded` 为 `True`，通常说明服务已起来。

## 6. 运行方式 B：idalib 无头模式

适合批处理或服务部署，不依赖 IDA GUI。

```powershell
uv run idalib-mcp --host 127.0.0.1 --port 8745 path\to\binary
```

多上下文隔离：

```powershell
uv run idalib-mcp --isolated-contexts --host 127.0.0.1 --port 8745 path\to\binary
```

## 7. 在 VS Code 中接入 MCP

如果你使用 VS Code MCP 功能，可在工作区放置 `.vscode/mcp.json`。

示例（HTTP 连接本机 IDA 插件 MCP）：

```json
{
  "servers": {
    "ida-pro-mcp": {
      "type": "http",
      "url": "http://127.0.0.1:13337/mcp"
    }
  }
}
```

保存后重载 VS Code，MCP 客户端应可发现该服务。

## 8. 常用开发命令

### 8.1 启动服务

```powershell
uv run ida-pro-mcp
uv run ida-pro-mcp --unsafe
uv run ida-pro-mcp --transport http://127.0.0.1:8744/sse
```

```powershell

```

### 8.3 测试

```powershell

```

## 9. 故障排查（高频）

### 9.1 `ida-pro-mcp` 命令找不到

原因：脚本不在 PATH。

解决：统一用 `uv run` 前缀执行，例如：

```powershell
uv run ida-pro-mcp --install
```

### 9.2 客户端连不上 `127.0.0.1:13337`

原因：IDA 插件服务未启动，或端口被占用。

排查：
- 在 IDA 内确认插件确实启动
- `Test-NetConnection 127.0.0.1 -Port 13337`

### 9.3 IDA Python 版本不匹配

现象：插件加载异常、导入失败。

解决：使用 `idapyswitch` 切换 IDA 使用的 Python，再重启 IDA。

### 9.4 路径有空格导致命令失败

解决：PowerShell 中路径统一加引号。

### 9.5 测试不稳定或个别断言差异

说明：不同 IDA/Hex-Rays 版本可能有轻微差异。

建议：
- 保留语义断言
- 对版本差异场景使用守卫判断或条件跳过
